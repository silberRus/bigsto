Как правильно указать участников в настройке прямого обмена с банками?

В настройке прямого обмена с банками фиксируется следующая информация:

  • выбирается "Организация";
  • задается "Банк", с которым планируется выполнять взаимодействие;

В зависимости от выбранного банка программа отображается набор необходимых параметров для заполнения.

Если выбран банк, параметры обмена с которым не известны программе, то в форме появляется возможность выбора варианта подключения к банковскому сервису.

Существует 3 варианта подключения:

- Прямое соединение с сервисом банка (асинхронно). Запрос данных и их получение разделены на отдельные процессы.

- Прямое соединение с сервисом банка (синхронно). При запросе данных программа всегда ожидает от банковского сервиса ответ. При данном варианте возможно 2 варианта авторизации: По сертификату и По логину и паролю.

Для данного варианта необходимо указать "Ресурсы банка для обмена данными" за закладке "Настройки обмена данными" - это адрес веб-сервера банка, а также адреса ресурсов для отправки и получения данных. Значения данных параметров выдает сам банк при заключении договора с клиентом.

- Через дополнительную обработку - используется дополнительная обработка для обмена с банком.

Для возможности использования данного варианта необходимо в программе включить использование дополнительных отчетов и обработок. Затем загрузить внешнюю обработку из файла и указать её в соглашении. Файл с обработкой предоставляет банк.

Важно. Для обмена электронными документами с конкретным банком необходимо ввести только одно соглашение по каждой организации. В рамках этого соглашения будут действовать настройки для всех банковских счетов организации.

Как настроить прямой обмен в асинхронном режиме?

При данном варианте формируются электронные документы и отправляются в банк без подписи. После отправки платежные документы появляются во внешнем клиент-банке. Чтобы платеж был исполнен его необходимо дополнительно подтвердить в клиент-банке. Для авторизации на ресурсе банка используется логин и пароль, для этого в форме Настройка ЭДО нужно:

  • выбрать программу банка "Прямое соединение с сервисом банка (асинхронно)".
  • ввести идентификатор организации
  • указать адрес сервера банка

Как настроить прямой обмен с использованием электронной подписи?

Для использования подписи электронных документов при прямом обмене с банками нужно:

  • в общих настройках программы включить возможность применения электронной подписи (ЭП);
  • выполнить общую настройку криптографии в системе;
  • выбрать вариант подключения "Прямое соединение с сервисом банка (синхронно)". 
  • указать Адрес сервера банка, Ресурсы для отправки и получения электронных документов
  • указать вариант авторизации - по сертификату подписи
  • в форме Настройки ЭДО в табличной части "Сертификаты ключа ЭП" указать элементы из списка "Сертификаты ЭП", а также настроить порядок следования "первой" и "второй" подписей организации на электронных документах.

Важно. Порядок следования элементов "Сертификатов ЭЦП" в списке определяет, кому принадлежит "первая", а кому "вторая" подписи. Если у организации только "первая" подпись на платежных поручениях, то необходимо указать только один сертификат ЭЦП в настройках соглашения.

Как настроить прямой обмен с банком с аутентификацией по логину и паролю?

При данном варианте формируются электронные документы и отправляются в банк без подписи. После отправки платежные документы появляются во внешнем клиент-банке. Чтобы платеж был исполнен его необходимо дополнительно подтвердить в клиент-банке. Для авторизации на ресурсе банка используется логин и пароль, для этого в форме Настройка ЭДО нужно:

  • выбрать программу банка "Прямое соединение с сервисом банка (синхронно)".
  • выбрать вариант авторизации "По логину и паролю".

 

Как настроить прямой обмен через дополнительную обработку ?

Для возможности использования данного варианта необходимо в программе включить использование дополнительных отчетов и обработок. Затем загрузить внешнюю обработку из файла и указать её в соглашении. Файл с обработкой предоставляет банк.

При использовании обмена через дополнительную обработку все функции криптографии, аутентификации и передачи данных выполняются средствами дополнительной обработки. Для использования обмена через дополнительную обработку необходимо:

  • в настройках программы включить использование дополнительных отчетов и обработок
  • добавить внешнюю обработку в справочник Дополнительные отчеты и обработки
  • выбрать загруженную обработку в настройке ЭДО
  • в форме настройки ЭДО с помощью кнопки "Загрузить сертификат из хранилища" добавить необходимые сертификаты подписи.
